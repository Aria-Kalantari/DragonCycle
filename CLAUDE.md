# Claude Code project instructions

## Source of truth

- Read `SPEC.md` before making architectural or gameplay changes.
- Use `Docs/DECISIONS.md` for approved deviations. Never silently contradict the spec.
- Work on the first unblocked item in `TASKS.md` unless the user names another task.

## Product constraints

- This is fully 3D dragon-vs-dragon combat for PC.
- Do not turn it into Souls-like dodge combat, a 2D fighter, or human sword combat.
- Both creatures can attack, defend, interrupt, grapple, and reposition at any time.
- No universal invulnerability roll.
- Stamina loss must reduce physical performance before reaching zero.
- Body-part damage must change capabilities and AI behavior.
- There are exactly three morphotypes: drake, wyvern, arch-serpent. Mythic lineages are content dressed over these; a new lineage never means a fourth body plan.
- Morphotypes must differ by anatomy and ecology, not only stats or VFX.
- Age, condition, and experience are independent variables, and aging is a curve rather than a buff.
- Wing loading trades speed against agility. A heavier-loaded morphotype is never also more agile.
- Takeoff is gated by launch mode, limb integrity, clearance, and stamina. Failure denies flight; it does not slow it.
- A committed stoop cannot be re-aimed mid-dive.
- Difficulty scales judgement, never health, damage, or reaction speed.
- Concealment is line-breaking and silhouette management, never invisibility.

## Engineering rules

- Target Unreal Engine 5.8.
- Prefer C++ for movement math, damage, data validation, save schemas, and reusable systems.
- Prefer Blueprint for assembly, tuning, animation/VFX/audio binding, and encounter content.
- Use Gameplay Ability System and Gameplay Tags for combat actions and state gating.
- Use data assets; avoid hard-coded morphotype-specific branches in shared systems.
- JSON in `Data/` is the authoring source; data assets are generated by the importer and hold runtime authority (ADR-006). The schema's part-id enum and `EDragonBodyPart` are one closed set — change them together.
- Every gameplay tag in SPEC 13.3 must exist in `Config/DefaultEngine.ini`.
- Keep classes small and components focused.
- Use `DragonCycle` as the C++ module name.
- Never edit generated Unreal folders: `Binaries/`, `DerivedDataCache/`, `Intermediate/`, `Saved/`.
- Unreal MCP is available for editor work when the plugin is enabled. Issue tool calls one at a time — invocations run serially on the game thread. Never assume it is connected; check.
- Editor work done through Unreal MCP is reviewed, tested, and committed like any other change. It is Experimental, unauthenticated, and localhost-only; do not build workflows that depend on its API staying stable.
- Never commit secrets, local absolute paths, or generated AI media unless explicitly selected.

## Workflow

1. Restate the task and identify affected systems.
2. Inspect existing files before editing.
3. Propose the smallest vertical implementation.
4. Implement with debug visibility.
5. Run focused tests and `python Tools/validate_data.py`.
6. Update `TASKS.md` and documentation.
7. Summarize changed files, tests, risks, and editor steps still required.

## Unreal conventions

- Prefix C++ classes with Unreal-required `A`, `U`, `F`, `E`, or `I`.
- Blueprint assets: `BP_`, data assets: `DA_`, abilities: `GA_`, effects: `GE_`, cues: `GC_`, montages: `AM_`, StateTrees: `ST_`, input actions: `IA_`, mapping contexts: `IMC_`.
- Public Blueprint APIs require `Category`, clear names, and comments/tooltips.
- Use `ensureMsgf` for recoverable developer errors and validation methods for assets.
- Avoid per-frame allocation and repeated full-world searches.
- Do not implement physics-critical behavior entirely in animation Blueprints.

## Quality gates

- The C++ project should compile after every coherent change.
- Add a test for non-trivial deterministic math.
- Emit telemetry from the single `CombatEvent` source. Never add a parallel analytics path.
- If a change would breach a falsification threshold in `Docs/TELEMETRY.md`, say so rather than retuning around it.
- Do not claim Unreal editor work is complete when only source files changed.
- Mark editor-only setup explicitly in the task summary.
- Generated Higgsfield output is reference, never final collision or skeletal data.
