# Unreal Editor bootstrap checklist

The source repository cannot create binary `.uasset` files. Complete these steps in Unreal Engine 5.8.

## Plugins

Enable and restart:

- Gameplay Ability System
- Gameplay Tags
- Gameplay Tasks
- Enhanced Input
- Niagara
- StateTree
- Gameplay StateTree
- Environment Query System
- Motion Warping
- Control Rig
- IK Rig
- Unreal MCP (plugin identifier `ModelContextProtocol`)
- All Toolsets (required — Unreal MCP implements no tools of its own)

## Unreal MCP setup

Experimental in 5.8, and the fastest way to make editor bootstrap agent-driven
rather than a manual checklist.

1. Edit > Plugins, enable **Unreal MCP** and **All Toolsets**, restart.
2. Edit > Editor Preferences > General > Model Context Protocol. Enable auto-start if you want the server up with the editor.
3. Run `ModelContextProtocol.GenerateClientConfig` in the editor console to write a Claude Code client config.
4. Launch Claude Code from the project root so it picks up the config and `CLAUDE.md` together.

The server binds to `127.0.0.1:8000/mcp` over HTTP and SSE. It is localhost-only,
has **no authentication layer**, and is not designed for remote use. Tool calls
run serially on the game thread, so overlapping calls are a mistake, not a
speed-up. Treat agent permissions like production credentials.

Everything it creates is still reviewed, tested, and committed like hand-made
work. Experimental means breaking changes before UE6.

## Collision channels

Add project trace/object channels:

- `DragonBody`
- `DragonAttack`
- `DragonWing`
- `DragonEnvironment`
- `BreathVolume`

Attacks should query `DragonBody`; body zones should ignore their owning actor.

## Gameplay tags

The tags in `SPEC.md` §13.3 are already declared in `Config/DefaultEngine.ini`; verify rather than re-create them. Add lineage, life-stage, experience, body-impairment, and attack tags as content expands.

## First map

- Use simple collision only.
- Add basin floor, two shelves, five pillars, one thermal trigger volume, and one cliff.
- Place scale references at 2 m, 10 m, and 30 m.
- Keep lighting simple until locomotion and camera are validated.

## First Blueprint child

Create `BP_Drake_Player` from `ADragonPawn` and assign:

- a proxy skeletal mesh;
- a species data asset generated by the importer (do not hand-author it);
- spring-arm socket/offset values;
- body hit-zone components per major bone;
- placeholder animation blueprint.
